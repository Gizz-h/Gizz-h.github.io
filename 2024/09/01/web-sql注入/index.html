这里考的是and与or运算符的优先级问题 and>or

利用这个可以构造查询语句

where (username !=’flag’ and id = ‘’)or(id=26 and’1’=’1’) limit 1

web183

like盲注

利用脚本进行盲注

  1. 字符串模板: {} 是一个占位符，它会被 format 函数替换为传递给它的参数。
  2. 构建payload: format(flag+j) 将 flag 变量和当前循环中的字符 j 拼接起来，然后替换掉字符串模板中的 {}。
  3. SQL注入: 拼接后的字符串被用作SQL查询的一部分，目的是让数据库返回那些密码字段以 flag 变量当前值加上新字符 j 开头的用户记录。
  4. 条件判断: 脚本检查服务器响应中是否包含特定的字符串（"$user_count = 1;"），如果包含，这意味着找到了正确的字符，因为SQL查询成功匹配了至少一个记录。
  5. 构建flag: 每次找到正确的字符时，该字符就被添加到 flag 变量中，逐步构建出完整的flag。

这里涉及两个知识点

  • md函数中的 md5(string,raw)`其中raw参数可选，且有两种选择
    • FALSE：32位16进制的字符串
    • TRUE：16位原始二进制格式的字符串
  • 在mysql里面进行布尔类型判断的时候
  • 在用作布尔型判断时，以1开头的字符串会被当做整型数。要注意的是这种情况是必须要有单引号括起来的，比如password=‘xxx’ or ‘1xxxxxxxxx’，那么就相当于password=‘xxx’ or 1 ，也就相当于password=‘xxx’ or true，所以返回值就是true。当然在我后来测试中发现，不只是1开头，只要是数字开头都是可以的。
    当然如果只有数字的话，就不需要单引号，比如password=‘xxx’ or 1，那么返回值也是true。（xxx指代任意字符）

利用这个知识点

$sql = “select count() from ctfshow_user where username = ‘admin’ and password= ‘ffifdyop’”;
$sql = “select count() from ctfshow_user where username = ‘admin’ and password= ‘’or’6�]��!r,��b’”;

将ffifdyop函数编码成’’or’6�]��!r,��b’”从而实现密码绕过 数字开头会被翻译成true 也就是通过了password

这里利用的是mysql的弱比较

在mysql中当出现以字母为开头的字符型数据在与数字型比较时，会将字符型数据转换成0再与数字比较

所以我们将username和password都设置为0 让用户名和密码都通过匹配即可
